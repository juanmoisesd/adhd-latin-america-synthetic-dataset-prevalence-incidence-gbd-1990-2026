# Security Policy

## 1. Reporting a Vulnerability
If you discover a security vulnerability within this repository, please send an email to the maintainers (see AUTHORS.md). We will acknowledge your report within 48 hours.

## 2. Integrity Verification
All releases are signed and includes BLAKE3/SHA-512 checksums. Users should verify these before use.

## 3. Supply Chain Security
The repository uses SLSA provenance and SBOM (Software Bill of Materials) to ensure the integrity of the development and build pipeline.
